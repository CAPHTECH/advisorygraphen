# 現行業務フロー・As-Is分析

## 分析概要

- 調査期間: 2026年3月（過去12ヶ月分のデータを対象）
- データソース: 経理部保管のExcel申請記録 2,340件
- 分析者: 業務改革推進室

---

## 現行プロセスフロー

```
[申請者]
  ↓ Excelに記入・領収書を印刷物に貼付
[申請者 → 直属上長]
  ↓ 社内メール送付または手渡し
[直属上長]
  ↓ 確認・承認（または差し戻し）
[上長 → 経理部]
  ↓ メールで回送
[経理部]
  ↓ 内容確認・仕訳・マスター表への転記
[経理部 → 経営管理システム]
  月次バッチで振込処理
```

---

## ボトルネック分析

### 各フェーズの所要時間（中央値）

| フェーズ | 所要時間（中央値） | 備考 |
|----------|-------------------|------|
| 申請作成 | 0.3日 | 申請日からExcel送付まで |
| **上長承認待ち** | **3.5日** | **最大ボトルネック** |
| 経理確認・転記 | 1.2日 | |
| **合計（申請→振込指示）** | **5.0日** | |

### 上長承認待ちの内訳

承認待ち3.5日のうち:
- **承認者が出張・外出中**: 2.1日分（全体の**60%**）
- 承認者が確認済みだが返信遅延: 0.8日
- その他（差し戻しによる手戻り含む）: 0.6日

**→ 承認遅延の主因は承認者の物理的不在であることがデータで確認される。**

承認者が社内にいる日の承認待ち平均: **1.1日**
承認者が出張中の日を含む申請の承認待ち平均: **6.8日**

### 差し戻し率と理由

- 全体差し戻し率: 14.2%（334件 / 2,340件）
- 差し戻し理由の内訳:
  - 領収書の添付漏れ: 47%
  - 金額・日付の転記ミス: 31%
  - 経費カテゴリの誤り: 15%
  - その他: 7%

### 経理処理工数

- 月次集計作業: 経理担当 2名 × 3日 = 6人日 / 月
- 主要作業: 各部門Excelからのデータ転記、仕訳確認、エラーチェック
- ヒューマンエラー発生率: 1.8%（月次集計での転記ミス）

## 改善インパクト試算

| 施策 | 期待効果 | 根拠 |
|------|----------|------|
| 代理承認機能の導入 | 承認待ち: 3.5日 → 1.2日 | 不在時の60%分が解消 |
| 自動集計・仕訳連携 | 月次工数: 6人日 → 1人日 | 手作業転記の排除 |
| 申請フォームの入力補助 | 差し戻し率: 14% → 8% | 入力ミス・漏れの減少 |

---

## 補足：代替仮説の検討

**「承認遅延の主因はプロセスの複雑さ（多段階承認）ではないか？」** という仮説を検討した。

データを確認すると：
- 単段階承認（上長のみ）の申請: 平均3.4日
- 二段階承認（上長→部門長）の申請: 平均3.7日

多段階承認の有無による差は0.3日にとどまる。承認フェーズの数よりも、承認者の在席状況の方が大きく影響していることが確認された。プロセス段数の削減では根本的な解決にならない。
