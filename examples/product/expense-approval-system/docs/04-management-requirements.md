# 経営・事業側の要望書

## 発行元

経営管理室（CFO 承認）

## 発行日

2026年3月28日

---

## 背景と目的

当社の経費精算業務は現行のExcel運用を10年以上継続しており、以下の経営課題が顕在化している。

1. **コンプライアンスリスク**: 申請内容の事後確認が困難で、不正・過誤の早期発見ができていない
2. **管理コスト**: 経理部門の経費処理に月6人日を要しており、スケールしない
3. **支出可視性の欠如**: 月次集計しか存在せず、週次・日次での予算管理が不可能
4. **承認の非効率**: 承認フローの停滞が申請者の不満と立替金長期化を招いている

---

## 要求事項

### 必須要件（Must Have）

1. **コンプライアンス強化**
   - 申請金額・頻度・カテゴリの異常値を自動フラグ
   - 上限金額超過、規程外カテゴリの申請を自動検出
   - 過去申請の監査証跡を3年分保持

2. **経理工数の大幅削減**
   - 弥生会計との自動連携（手動転記の撤廃）
   - 月次集計の自動生成

3. **経営レポーティングの強化**
   - リアルタイムの支出ダッシュボード（週次・日次ビュー）
   - 部門別・カテゴリ別の予算実績比較

### 重要要件（Should Have）

4. **承認フローの改善**
   - 承認者不在時の代理承認または自動エスカレーション
   - 承認期間の目標: 現行5日 → 1日以内

5. **申請操作の改善**
   - 申請ステータスのリアルタイム確認
   - 領収書のデジタル添付（OCRによる自動入力があればなお良い）

### 任意要件（Nice to Have）

6. **モバイルアプリ**
   - 優先度は低い。現場からの要望があれば検討する

---

## 制約事項

| 制約 | 内容 |
|------|------|
| 予算上限 | 初期開発費 + 年間ランニングコスト 合計3,000万円以内 |
| リリース期限 | 次年度Q1（2027年4月）開始に間に合うこと |
| 連携必須システム | 弥生会計（現行バージョン：弥生会計 24） |
| データ移行 | 過去3年分の申請データを移行すること |

## 成功指標

| KPI | 現状 | 目標 |
|-----|------|------|
| 承認リードタイム（中央値） | 5日 | 1日以内 |
| 経理工数（月次集計） | 6人日 | 1人日以内 |
| 差し戻し率 | 14% | 8%以下 |
| 申請漏れ率 | 不明（測定不可） | 測定可能にする |
| 不正・異常値検出件数 | 0件（検出手段なし） | 月次で報告可能にする |

## 経営からの補足メモ

> リアルタイムダッシュボードについては、競合他社の導入事例を見ていると「あれば使う」機能に留まっており、実際の不正抑止効果については定量的なエビデンスが少ない。過大な期待を持たず、まず承認フロー改善と連携自動化を優先することを推奨する。
>
> モバイルアプリの優先度を「任意」にしたのは、当社の外勤比率が全社員の約20%であるため。投資対効果を確認してから判断したい。
